ライブマイグレーションの解説

稼働中のシステムを止めることなく、別の物理サーバーへ移動させる仮想化技術について解説します。

# 1. 用語の意味（結論）

👉 稼働中の仮想マシンを停止させることなく、別の物理サーバへ移動させる技術

（つまり、「サービスを止めずに引っ越しする技術」）

# 2. なぜ必要なのか（背景）

クラウドや仮想化環境では、以下のような課題が発生します。

  • 特定のサーバにアクセスが集中して負荷が高すぎる
  • ハードウェアの保守・点検のために電源を落としたい
  • 故障の予兆があるサーバから安全な場所へ逃がしたい
🤔
しかし、サーバを止めると…
Webサービスが停止し、利用者に迷惑がかかってしまいます。
そこで「ライブマイグレーション」の出番！

# 3. 仕組み（どうやって移動する？）

VM
物理サーバ A
（移動元）
Network Copy
メモリを同期
↓
一瞬で切り替え
VM
物理サーバ B
（移動先）

# 4. 具体例（ECサイトの場合）

状況：サーバAの負荷が高すぎる！

システム側の動き

  • ECサイトの仮想マシンをサーバAからBへ移動開始
  • データの差分をバックグラウンドで同期
  • 完了直前に一瞬だけ処理を止めて切り替え

利用者側の体験

  • 買い物カゴの中身はそのまま
  • ログイン状態も維持される
  • 画面が固まったりエラーになったりしない

# 5. 他の用語との違い（試験対策）

「コールドマイグレーション」との違いを理解しておきましょう。

項目 ライブマイグレーション コールドマイグレーション
仮想マシン 稼働中（ON） 停止（OFF）
サービス停止 ほぼなし あり
利用シーン 24時間稼働・日中 夜間メンテナンスなど

# 6. ITパスポート試験でのポイント

  • • 「仮想化技術」「クラウド」の分野で出題される
  • • 「可用性（Availability）」を高める技術の一つ
  • • キーワードは「稼働中のまま」「停止せず」
Exam Example

Q. 稼働中の仮想サーバを、利用者に影響を与えず別の物理サーバへ移動する技術はどれか？

A. ストリーミング
B. ライブマイグレーション
C. クラスタリング
D. プラグアンドプレイ
👉 正解：B
「稼働中のまま」「移動」というキーワードが出たらライブマイグレーションです。

まとめ（一言）
