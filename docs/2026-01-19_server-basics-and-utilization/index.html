#サーバー #Webアプリ #クラウド #開発環境

1. サーバーの基本概念

定義

ネットワーク上で他のコンピュータ（クライアント）にサービスやデータを提供するコンピュータやソフトウェアのこと。

必ずしも高性能なハードウェアである必要はなく、「役割」を指す言葉です。

例え：料理を作る厨房

  • CLIENT → お客さん（注文する人）
  • SERVER → 厨房（料理を作って出す）
  • SERVICE → 料理（データや機能）

2. サーバーの種類と例

種類 主な役割 代表的なソフトウェア例
Webサーバー Webページの配信 Apache, Nginx
メールサーバー メールの送受信 Postfix, Exchange
DBサーバー データベース管理・提供 MySQL, PostgreSQL
ファイルサーバー ファイルの保管・共有 Samba, FTP

Point: サーバーは「データの倉庫」ではなく、「提供するサービス」によって役割が決まります。

5. Webアプリ自作の環境

学習・テスト用

自分のPC (Local)

最も手軽。XAMPPやNode.jsをインストールして動かす。ネット公開には不向き。

> Node.js
> XAMPP / MAMP
低コスト・IoT

小型PC

Raspberry Piなど。省電力で常時稼働させやすい。Linuxの勉強にも最適。

> Raspberry Pi
> Ubuntu Server
本番公開用

クラウド (Cloud)

AWSやHerokuなど。インターネット上のどこからでもアクセス可能。

> AWS / GCP
> Heroku / Render

Summary

サーバーは「ネットワーク経由でサービスを提供するもの」。
まずは手元のPCで環境を作り、慣れたらクラウドで世界に公開しましょう。
