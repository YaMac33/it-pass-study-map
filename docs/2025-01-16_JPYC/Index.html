スマホアプリ不要？マイナンバーカードで「JPYC」決済が可能に

2026年1月16日 #Web3 #FinTech

本日（2026年1月16日）、日本の決済シーンにおいて非常に興味深いニュースが飛び込んできました。 三井住友カードとマイナウォレット社が、「マイナンバーカードを活用したステーブルコイン決済」の実証実験を開始するというものです。

Web3やブロックチェーン技術が、いよいよ私たちの「物理的な日常」に溶け込み始めたことを実感させるニュースです。今回はこの仕組みの何が画期的なのか、技術的な視点を交えて解説します。

ニュースの核心：3者の連携

このプロジェクトは、主に以下の3つの要素が組み合わさって成立しています。

  • マイナンバーカード：認証とウォレットの鍵として利用
  • 三井住友カード (stera)：既存の決済端末インフラを提供
  • JPYC：決済に使われる日本円連動型ステーブルコイン
2026年1月23日より、福岡市のプロバスケットボール試合会場などで第一弾の実証実験が開始されます。

なぜ「アプリ不要」が革命的なのか

これまでの「スマホ決済」や「暗号資産決済」には、必ずと言っていいほど「専用アプリのインストール」や「複雑なウォレット操作」というハードルが存在しました。

しかし、今回の仕組みは違います。
ユーザーが行うのは、店頭の決済端末にマイナンバーカードをかざすだけ。

仕組みの裏側 (JPKI)

この簡便さを支えているのは、マイナンバーカードのICチップに搭載されている公的個人認証サービス（JPKI）です。

  1. ユーザーは事前にウォレットとマイナンバーカードを紐付ける。
  2. 店頭でカードをかざすと、JPKIによる本人確認が走る。
  3. 認証が成功すると、紐付いたブロックチェーンウォレットから JPYC が支払われる。

つまり、高齢者や子供など、スマートフォンの操作に不慣れな層であっても、物理カードさえ持っていれば最新のデジタル通貨決済が可能になるのです。

三井住友カードとJPYCの役割

インフラとしての三井住友カード

クレジットカード大手の三井住友カードは、この実験のために新しい専用端末を配るわけではありません。 既存の加盟店に普及している高機能端末 「stera terminal」 を活用します。

Androidベースのstera端末に機能を追加することで、「マイナンバーカードを読み取り、ブロックチェーン決済を行う」という離れ業を、既存のインフラ上で実現しています。

決済通貨としてのJPYC

決済には 1 JPYC = 1円 で取引されるステーブルコインが使用されます。 ビットコインのような価格変動リスクがなく、銀行システム（全銀ネット）を介さずに24時間365日送金できる点が強みです。 

// 概念的な決済フロー
async function processPayment(cardData, amount) {
    // 1. マイナンバーカードのJPKIで本人認証
    const user = await authenticateWithJPKI(cardData);
    
    if (user.isAuthenticated) {
        // 2. ブロックチェーン上でJPYCを送金
        const txHash = await sendJPYC(user.walletAddress, storeAddress, amount);
        return txHash;
    }
}

今後の展望

この「物理カード × ブロックチェーン」の組み合わせは、決済だけにとどまりません。

  • インバウンド対応：外国人の持つUSDCなどと連携すれば、両替不要の決済が可能に。
  • 地域通貨：自治体が配布するデジタル給付金を、カード一枚で街のお店で使えるように。

「Web3は難しい」という常識を覆す、非常に日本的かつ実用的なアプローチだと言えるでしょう。1月23日からの実証実験の結果に注目が集まります。
