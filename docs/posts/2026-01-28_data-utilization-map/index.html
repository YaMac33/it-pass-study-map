「DWH」「ETL」「データマート」…データ分析に関する用語はたくさんありますが、これらはバラバラに存在するわけではありません。
データの「流れ」に沿って全体像を捉えると、それぞれの役割が一気に明確になります。

1行要約
「OPAL」という流れの中で、「ETL/ELT」で運び、「DWH/データレイク」に貯め、「データマート/BI」で使う。

2. 用語の一言解説

試験対策として、それぞれの「役割」を一言で定義します。

用語 役割（一言で） 試験でのポイント
OPAL 全体の流れ・概念 「元データ→処理→分析→出力」のプロセス全体を指す。
ETL 整えてから運ぶ DWHに入れる前に形式を統一する。
ELT 運んでから整える ビッグデータやクラウドで主流。まずデータレイクに放り込む。
DWH 整理された倉庫 「時系列」「統合」「消さない」。全社分析用。
データレイク 生の貯水池 画像やログなど「非構造化データ」もそのまま保存できる。
データマート 特定の売場 DWHから「営業部用」など特定目的のために切り出したもの。
BI 可視化ツール 人間が見て判断するためのグラフやダッシュボード。

3. 試験での見分け方

問題文に含まれるキーワードで、どの箱の話かを見抜きましょう。

  • 「経営判断のために全社のデータを統合」→ DWH
  • 「特定の部門が利用するために抽出」→ データマート
  • 「形式の異なるデータを統一して格納」→ ETL
  • 「大量の生データをそのまま蓄積」→ データレイク

まとめ（暗記用）

最後は、このフレーズで位置関係を頭に入れましょう。

OPAL は「流れ」 ETL / ELT は「方法」 レイク・DWH は「場所」 マート は「小分け」 BI は「出口」
