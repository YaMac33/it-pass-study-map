ビットごとの論理積とマスクの役割

  • #論理積
  • #マスク
  • #2進数

「8ビットのデータXと00001111の論理積をとる」
ITパスポートや基本情報技術者試験でよく見るこの手の問題。難しそうに見えますが、実は計算問題ではなく「型抜き」のパズルだと考えれば一瞬で解けます。

1行要約
論理積（AND）とマスクを使った演算は、データの「欲しい部分だけを残して、要らない部分を消す」ためのフィルター処理である。

1. 論理積（AND）の基本ルール

まずはシンプルに「1桁（1ビット）」だけの計算ルールを確認します。
論理積（AND）は、日本語で言うと「両方とも1なら1」です。

1 AND 1 = 1
（残る）
1 AND 0 = 0
0 AND 1 = 0
0 AND 0 = 0
（消える）

2. 「マスク」とは何か？

この性質を利用して、8桁（8ビット）のデータから特定の部分だけを取り出す操作を「マスク（Mask）」と呼びます。

クッキーの型抜きをイメージしてください。
「1」は穴が空いている部分（通す）、「0」は塞がっている部分（消す）です。

視覚イメージ：下位4ビットを取り出す

3. 実際の計算（筆算）

試験では以下のように、桁をそろえて縦に計算します。

データX10101010
マスク00001111
AND00001010

計算する必要すらありません。
「マスクが0のところは0、1のところはそのまま降りてくる」と見れば一瞬です。

4. 何に使われるのか？（実務での用途）

ただの計算遊びではありません。実務では「フラグ管理」などで頻繁に使われます。
例えば、1つの数字の中に「権限情報」などを詰め込んでいる場合です。

ビット位置 意味 値が 1 なら...
上位4ビット 部署コード （部署の種類を表す）
下位4ビット 権限フラグ 管理者権限あり

このデータから「権限情報（下位4ビット）」だけを取り出してチェックしたい場合に、00001111 でマスクをかけます。

まとめ

ITパスポート試験で「論理積」「AND」が出たら、以下の合言葉を思い出してください。

  • AND ＋ 1 ＝ そのまま残す（型抜き）
  • AND ＋ 0 ＝ 強制的に0にする（消す）
  • 00001111 ＝ 下位4ビットを取り出すマスク
