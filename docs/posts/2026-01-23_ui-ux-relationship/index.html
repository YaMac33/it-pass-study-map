UIとUXの関係性のまとめ

  • #UI
  • #UX
  • #GUI
  • #CUI
  • #デザイン

「UI」と「UX」。どちらもシステム開発やデザインの現場で頻繁に使われますが、それぞれの境界線や関係性を正しく説明できるでしょうか？
さらに「GUI」や「CUI」といった用語も加わると混乱しがちです。今回はこれらの用語を整理し、一目で分かる関係図にまとめました。

1行要約
UI（接点）はUX（体験）の一部であり、そのUIの種類としてGUI（視覚操作）やCUI（文字操作）が存在する。

1. 基本用語の意味

まずは各用語の定義と具体例を整理します。

UI User Interface / ユーザーインターフェース
ユーザーとシステム（機械・アプリ）がやり取りするための「窓口・接点」のこと。画面、ボタン、入力フォームなど、操作に関わるすべてを指します。
GUI Graphical User Interface
UIの一種。アイコンやボタンなど、「視覚的な画像」を用いてマウスやタップで操作する方式です。
例：Windowsのデスクトップ、スマホのホーム画面
CUI Character User Interface
UIの一種。キーボードで「文字（コマンド）」を入力して操作する方式です。プログラマやサーバ管理者がよく使います。
例：Windowsのコマンドプロンプト、Linuxのターミナル
UX User Experience / ユーザーエクスペリエンス
UIを通じてユーザーが得る「体験全体」や「感情・満足度」のこと。「使いやすい」「楽しい」「ストレスがない」といった感覚を含みます。

2. 関係性の図解

これらの用語はバラバラに存在するのではなく、明確な包含関係（親子関係）があります。

UX（ユーザー体験）
満足度、感情、使いやすさ
UI（インターフェース）
システムとの接点・操作手段
GUI
視覚的操作
CUI
文字操作

図の読み解き方

  1. UIはUXの一部である： UI（見た目や操作性）が優れていることは、良いUX（体験）を生むための重要な要素です。しかし、UXはUIだけでなく「サービスの信頼性」や「サポートの質」なども含みます。
  2. UIの中にGUIとCUIがある： 「どうやって操作するか」という手段の違いとして、グラフィカルなGUIと、文字ベースのCUIが存在します。

3. 覚え方のコツ

試験や実務で混同しないためのポイントです。

  • UI ＝ 「接点」（画面そのもの）
  • GUI / CUI ＝ 「UIの形式」（絵か、文字か）
  • UX ＝ 「感想」（使ってみてどうだったか）

「UIデザインは綺麗だけど、使い勝手が悪くてUXは最悪だ」というような表現が成り立つことを覚えておきましょう。
