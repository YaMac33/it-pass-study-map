ウォーターフォール、スパイラル、アジャイルの違い

  • #開発手法
  • #ウォーターフォール
  • #アジャイル
  • #スパイラル
  • #プロジェクト管理

システム開発のプロジェクトを成功させるには、プロジェクトの性質（規模や変更の有無）に合った「進め方（開発モデル）」を選ぶ必要があります。
代表的な3つの手法「ウォーターフォール」「スパイラル」「アジャイル」の違いを、図解で整理します。

1行要約
ウォーターフォールは「一直線（計画重視）」、スパイラルは「リスク評価の繰り返し（慎重）」、アジャイルは「小刻みな改善（スピード重視）」。

2. 各手法の特徴詳細

① ウォーターフォールモデル

工程を「要件定義→設計→実装→テスト」と順番に進め、原則として後戻り（手戻り）をしない手法です。

  • メリット： スケジュールや予算の管理がしやすい。品質を確保しやすい。
  • デメリット： 途中で仕様変更があると、修正コストが莫大になる。
  • 向いている： 大規模システム、基幹システムなど「仕様が固まっている」案件。

② スパイラルモデル

システムを独立性の高いサブシステムに分割し、それぞれの開発サイクル（設計→実装→評価）を螺旋階段を登るように繰り返す手法です。

  • 特徴： サイクルごとにプロトタイプ（試作）を作り、ユーザーの評価を受けてから次に進む。
  • メリット： 早期にフィードバックが得られ、手戻りのリスクを減らせる。
  • 向いている： 未知の技術を使う場合や、要件が不明確な案件。

③ アジャイル開発（XP / スクラム）

「イテレーション（反復）」と呼ばれる短い期間（1〜4週間）で、「計画→実装→テスト」を繰り返し、少しずつ機能をリリースしていく手法です。

  • 特徴： 最初から完璧な計画を立てず、変化に対応することを優先する。
  • メリット： 顧客の要望変化に柔軟に対応でき、動くものを早く提供できる。
  • 向いている： Webサービス、スマホアプリ、スタートアップなど。

3. 比較まとめ表

試験対策として、それぞれの重視するポイントを整理します。

項目 ウォーターフォール スパイラル アジャイル
進め方 一直線 反復（リスク重視） 反復（速度重視）
変更対応 弱い（困難） 中程度 強い（柔軟）
重視点 計画・文書 品質・リスク スピード・変化
成果物 最後に完成 徐々に完成 すぐ動く

4. 試験でのひっかけ対策

問題文のキーワードで即答できるようにしましょう。

  • 「工程を順番に」「後戻りしない」→ ウォーターフォール
  • 「試作を繰り返す」「リスクを最小化」→ スパイラル
  • 「短い期間（イテレーション）」「仕様変更に柔軟」→ アジャイル

まとめ（暗記用）

開発手法は「変更へのスタンス」の違いです。

WF ＝ 一気通貫（変わらない前提） SP ＝ リスクぐるぐる（慎重に進む） AG ＝ 小刻み改善（変わる前提）
