TCP/IPのポート番号の役割

  • #TCP/IP
  • #ポート番号
  • #ネットワーク
  • #識別
  • #アプリケーション

ネットワーク通信において、「IPアドレス」が宛先のコンピュータを特定するものであることは有名です。
では、「ポート番号」は何のためにあるのでしょうか？

1行要約
ポート番号は、そのコンピュータの中にある「どのアプリケーション（サービス）」にデータを渡すかを識別するための番号である。

1. 「住所と部屋番号」で理解する

インターネット上の通信は、郵便や宅配便に例えると非常に分かりやすくなります。

宛先情報
IP：203.0.113.1
Port：80
➜
住所：203.0.113.1
80号室 Webサイト
(HTTP)
25号室 メール
(SMTP)
22号室 リモート
(SSH)
空室

IPアドレスで「建物」まで届き、ポート番号で「部屋（アプリ）」を指定する。

もしポート番号がなかったら、コンピュータまではデータが届いても、Webブラウザに渡すべきかメールソフトに渡すべきかが分からなくなってしまいます。

2. ネットワークの4つの識別子

試験では、以下の4つの用語が「何を識別するものか」という観点でよく問われます。

MACアドレス
同一LAN内の機器（NIC）を識別する。
（イメージ：隣の家への回覧板）
IPアドレス
ネットワーク越しの通信先の機器（ホスト）を識別する。
（イメージ：世界中の住所）
ポート番号
その機器の中のアプリケーション（サービス）を識別する。
（イメージ：マンションの部屋番号）
ESSID
Wi-Fiのネットワーク名を識別する。
（イメージ：接続するWi-Fiの名前）

3. よくある「ひっかけ問題」

ITパスポート試験でよくある間違いのパターンです。

  • ❌ 「ポート番号は、LANカード（NIC）を識別する番号である」
    → 誤： それはMACアドレスの説明です。
  • ❌ 「ポート番号は、接続するプロバイダを識別する番号である」
    → 誤： プロバイダの識別ではありません。

まとめ

以下の対応関係を覚えておけば、この分野の問題は確実に解けます。

  • IPアドレス ＝ 住所（建物）
  • ポート番号 ＝ 部屋番号（アプリ）

Webを見るときは「80番（または443番）の部屋」をノックしている、というイメージを持っておきましょう。
