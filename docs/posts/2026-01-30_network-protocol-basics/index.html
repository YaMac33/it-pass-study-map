コンピュータ同士が通信を行うためには、共通のルール（プロトコル）が必要です。
このルールは、役割ごとに7つの階層に分かれています。これをOSI参照モデルと呼びます。

1行要約
プロトコルとは「通信の約束事」であり、OSI参照モデルはそれを「物理的な信号」から「アプリの操作」まで7段階に整理したもの。

1. OSI参照モデルと主要プロトコル

下層（ハードウェア寄り）から上層（ソフトウェア寄り）へと積み上がっています。

2. 各層の代表的なプロトコル

試験によく出るプロトコルを、役割ごとに整理しました。

アプリケーション層
  • HTTP / HTTPSWebサイトの閲覧
  • SMTP / POP3メールの送信・受信
  • FTPファイルの転送
  • DNSドメイン名とIPの変換
トランスポート層
  • TCP信頼性重視（Web, メール等）
  • UDP速度重視（動画, 音声）
ネットワーク層
  • IP宛先（住所）への配送
  • ICMP通信確認（Pingなど）
  • ARPIPからMACアドレスを調べる
下位層（物理・リンク）
  • Ethernet有線LANの規格
  • IEEE 802.11無線LAN（Wi-Fi）
  • Bluetooth近距離無線

3. プロトコルの組み合わせ（カプセル化）

これらは単独で動くのではなく、積み木のように組み合わせて使われます。
例えば、安全なWeb閲覧（HTTPS）は以下のように構成されています。

HTTP (Web)
TLS (暗号化)
TCP (確実性)
IP (宛先)
Ethernet (伝送)
➡
HTTPS通信
安全なWeb閲覧

アプリがデータを渡し、下の層が封筒に入れ、さらに下の層が宛名を書き…というバケツリレーが行われています。

まとめ

  • 階層化のメリット： 上の層（アプリ）は、下の層（回線の種類など）を気にせずに開発できる。
  • 重要プロトコル： WebならHTTP、住所ならIP、信頼性ならTCP。
