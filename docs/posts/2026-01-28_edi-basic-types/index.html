EDIの基本と種類

  • #EDI
  • #電子取引
  • #業務効率化
  • #VAN
  • #Web-EDI

企業間の受発注や請求などの取引を、紙やFAXではなく、ネットワーク経由のデータ交換で行う仕組みが「EDI（電子データ交換）」です。
ペーパーレス化だけでなく、手入力ミスの削減や処理スピード向上など、業務効率化の切り札となります。

1行要約
EDIとは、異なる企業同士が、標準的なフォーマットを使って、専用回線やインターネットで商取引データを自動交換する仕組みのこと。

1. EDIの仕組み（データの流れ）

FAXや電話と違い、人間が介在せずにシステム同士でデータをやり取りします。

注文データが自動で相手のシステムに登録されるため、入力作業が不要になります。

2. EDIの2つの種類（VAN型 vs Web-EDI）

EDIには、昔ながらの「専用線」を使うタイプと、現代主流の「インターネット」を使うタイプがあります。

VAN型 EDI

特定業者が運営する専用ネットワーク（VAN）を使う。
特徴： 安全だがコストが高い。
規格： JCA手順、全銀手順など（レガシー）。

Web-EDI

インターネット回線を使う。
特徴： 安価で導入しやすい。ブラウザで操作可能。
規格： XML、EDIINTなど。

3. 比較まとめ表

試験対策として、それぞれの特徴を整理します。

項目 VAN型 (従来) Web-EDI (現代)
通信回線 専用線・電話回線 インターネット
コスト 高い 安い
導入ハードル 高い（専用機器が必要） 低い（PCとブラウザでOK）
セキュリティ 非常に高い（閉域網） 対策が必要（SSL/TLS等）

4. 標準規格について

異なる企業同士でデータをやり取りするためには、「データの並び順（フォーマット）」などのルール（プロトコル）を合わせる必要があります。

  • 国内規格： JCA手順、全銀手順（古いが金融などで現役）、JX手順（流通BMS）。
  • 国際規格： UN/EDIFACT（国連制定）、XML/UBL。

最近は、柔軟性の高いXMLを用いたデータ交換が増えています。

まとめ

  • EDI： 電子データ交換。業務効率化とペーパーレス化を実現。
  • VAN型： 専用線。安全だが高い。
  • Web-EDI： インターネット。安くて手軽。

「電話やFAXをやめて、ネットでデータを自動連携する仕組み」と覚えておけばOKです。
