ホスト名とドメイン名の理解

  • #DNS
  • #ホスト名
  • #ドメイン名
  • #FQDN
  • #ITパスポート

WebサイトのURLやメールアドレスで必ず目にする文字列。
「どこからどこまでがドメイン名で、どれがホスト名？」と迷うことはありませんか？

ITパスポート試験では、これらの区切り位置と役割を正しく理解しておく必要があります。

1行要約
ホスト名は「特定のコンピュータの名前（表札）」、ドメイン名は「所属する組織やグループ（住所）」、両方合わせたものがFQDN。

1. 全体像：FQDNの構造

まずは全体像（FQDN：Fully Qualified Domain Name）を分解してみましょう。

www.example.co.jp
ホスト名 機器の名前
ドメイン名 組織・グループ

※メールの場合：mail.google.com

2. それぞれの役割

① ホスト名（Host Name）

ネットワーク内にある「1台の具体的な機器（サーバー）」を識別する名前です。
「Webサーバーだから www」「メールサーバーだから mail」と付けるのが慣習ですが、実は server01 でも何でも構いません。

  • 役割： ドメイン（会社）の中にある、特定のコンピュータを指名する。
  • イメージ： 家の「表札」や、会社の中の「個人名」。

② ドメイン名（Domain Name）

インターネット上での「住所の範囲（組織）」を表します。
右側から順に大きな分類（国→種類→組織名）へと絞り込まれていく階層構造になっています。

  • 役割： インターネット上の管理エリアを区切る。
  • イメージ： 「東京都 港区 〇〇株式会社」という住所。

3. 階層構造で見る関係性

ドメインツリーとして見ると、ホスト名は一番下の「葉っぱ」にあたります。

jp (日本)
co (企業)
example (組織名)
www
Webサーバ
mail
メールサーバ
ftp
FTPサーバ

example.co.jp という会社の中に、www くんや mail くんがいるイメージです。

4. 比較まとめ

試験対策として、違いを表で整理します。

項目 ホスト名 ドメイン名
指すもの 個別の機器・サーバ 組織・ネットワーク
範囲 狭い（点） 広い（面）
例 www, mail google.com, co.jp
DNSの役割 これらを組み合わせてIPアドレスに変換する

まとめ

  • ホスト名： 「誰？」（個別のマシン名）
  • ドメイン名： 「どこ？」（組織やグループの住所）

URLを見るときは、「一番左のドットまでがホスト名、それより右がドメイン名」と区切って読む癖をつけましょう。
