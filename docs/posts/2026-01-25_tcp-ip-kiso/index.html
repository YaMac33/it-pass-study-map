TCP/IPの基礎とアプリケーション

  • #TCP/IP
  • #通信プロトコル
  • #ネットワーク
  • #アプリケーション層
  • #コンピュータ

インターネットを利用する際、私たちはWeb閲覧やメール、ファイル転送など様々なアプリケーションを使っています。
これらは一見バラバラのソフトに見えますが、実はその裏側では共通の通信ルール（プロトコル）が動いています。

1行要約
どんなアプリケーション（Web・メール等）であっても、通信の土台として共通に使われているのが「TCP/IP」である。

1. 通信の階層構造（イメージ）

ネットワーク通信は「役割分担」の階層構造になっています。
一番上にあるのが、私たちが直接使う「アプリケーション層」。その下で通信の実務をこなしているのが「トランスポート層（TCPなど）」と「インターネット層（IP）」です。

アプリケーション層（用途ごとに違う）
HTTP Web
SMTP メール送信
POP メール受信
FTP ファイル
⬇ 共通利用
TCP / IP
トランスポート層 ＋ インターネット層
「データを分割して、宛先まで確実に届ける」
基本的な通信機能

2. アプリケーション層プロトコルの整理

試験では「どのプロトコルが何をするものか」が頻出です。代表的なものを整理します。

HTTP / HTTPS
WebブラウザとWebサーバの間で、HTMLなどのデータをやり取りする。HTTPSは暗号化版。
SMTP
メールを送信・転送する。（相手のサーバまで届ける）
POP / IMAP
メールサーバからメールを受信する。
FTP
ファイルを転送する。（File Transfer Protocol）
NTP
時刻を合わせる。（Network Time Protocol）

3. よくある試験問題のパターン

ITパスポート試験で、以下のような問題が出たときの考え方です。

問：Webや電子メールなど様々なアプリケーションで共通に使われ、基本的な通信機能を実現するプロトコルはどれか。
ア：SMTP　イ：FTP　ウ：TCP/IP　エ：POP

解き方のコツ

  1. 選択肢を見る。ア、イ、エはすべて「特定のアプリ専用」のプロトコル。
  2. 問題文の「共通に使われる」「下位にあり」「基本的な通信機能」というキーワードに注目。
  3. アプリの土台となる「ウ：TCP/IP」が正解。

まとめ

  • アプリケーション層（HTTP, SMTP等）：用途によって使い分ける「道具」。
  • TCP/IP：それら全ての道具を動かすための「インフラ（土台）」。

「アプリはいろいろあるけど、足回りは全部TCP/IPに任せている」という構造を理解しておきましょう。
