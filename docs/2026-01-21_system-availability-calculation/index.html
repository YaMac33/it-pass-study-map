システム稼働率の計算手順：Web並列・DB直列の攻略法

  • #システム
  • #稼働率
  • #計算問題
  • #Webサーバ
  • #基本情報技術者

ITパスポートや基本情報技術者試験で必ずと言っていいほど出題されるのが「システム稼働率」の計算問題です。特に「Webサーバ2台（並列）＋DBサーバ1台（直列）」という構成は定番中の定番です。

公式を丸暗記するよりも、システムが動く仕組み（直列・並列）のイメージを持つことで、計算ミスを防ぐことができます。本記事では、以下のモデルケースを使って計算手順を解説します。

【例題モデル】
・Webサーバ2台（稼働率 各0.8）が並列接続
・その後ろにDBサーバ1台（稼働率 0.9）が直列接続
このシステム全体の稼働率を求めよ。

システム構成の可視化

まずは文章で書かれた構成を図にしてみましょう。並列部分は「どちらか片方でも動いていればOK」、直列部分は「すべて動いていないとダメ」という意味です。

Web A 0.8
Web B 0.8
➞ 直列
DB Server 0.9

Web層（並列）の結果と、DB層（単体）を掛け合わせるのがポイント

計算ステップ①：並列部分（Web層）

並列接続されたWebサーバの稼働率を求めます。並列システムの稼働率計算における鉄則は、「止まる確率（故障率）」から考えることです。

思考プロセス

  1. Webサーバが単体で停止する確率は？
    1 - 0.8 = 0.2
  2. 「2台とも」同時に停止する確率は？
    0.2 × 0.2 = 0.04
  3. じゃあ、「少なくとも1台は動いている」確率は？
    1 - 0.04 = 0.96

これで、Webサーバ部分を「稼働率 0.96 の1つの大きなサーバ」として扱えるようになりました。

計算ステップ②：全体（直列）の計算

次に、先ほど求めたWeb層の稼働率と、DBサーバの稼働率を繋げます。直列接続（WebもDBも両方動く必要がある）なので、単純な掛け算になります。

思考プロセス

  1. Web層全体の稼働率は 0.96
  2. DBサーバの稼働率は 0.9
  3. 両方が動いている確率は？
    0.96 × 0.9 = 0.864

試験対策：四捨五入の罠に注意

計算自体は単純な掛け算ですが、試験では「端数処理」で引っ掛け問題が出ることがあります。

例えば、「小数第3位を四捨五入して求めよ」という指示があった場合：

  • 計算結果：0.864
  • 小数第3位（4）を四捨五入
  • 回答：0.86

「小数第3位を四捨五入」＝「小数第2位まで求める」という意味です。問題文の指示をよく読み、最後の1桁で点数を落とさないように注意しましょう。

まとめ

システム稼働率の計算は、手順をパターン化すれば怖くありません。

  1. まず、並列部分を「1 - (停止率の積)」で計算し、1つの塊にする。
  2. 次に、すべての要素を直列（掛け算）でつなぐ。

この手順を守れば、Webサーバが3台になっても、DBが冗長化されても、同じロジックで解くことができます。
