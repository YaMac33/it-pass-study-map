プロジェクト統合マネジメントの概要

プロジェクト全体を俯瞰し、計画から実行、終結までを統括・調整する重要な役割について解説します。

# 1. 定義（結論）

👉 プロジェクト全体を横断的にまとめ、計画〜実行〜終結を一貫して調整・統制するマネジメント

（Project Integration Management）

# 2. ITパスポート試験での重要ポイント

  • • 全体をまとめる“司令塔”的な管理
  • • 他の管理（スコープ・コスト・品質など）を統合する
  • • プロジェクトマネージャの最重要役割
  • • キーワードは「調整・意思決定・全体最適」

📌 ポイント：個別管理ではなく「全体のつじつま合わせ」が本質です。

# 3. なぜ「統合」が必要か（背景）

プロジェクトでは「スコープ（範囲）」「スケジュール（期間）」「コスト（予算）」「品質」など、複数の要素が同時に動きます。これらは互いに影響し合う（トレードオフの関係にある）ため、バランスを取る必要があります。

よくある対立：例1
期間短縮したい → コスト（残業代など）が増える
よくある対立：例2
コスト削減したい → 品質が下がる恐れ
👉 これらを調整するのが「統合マネジメント」の役割

# 5. 主な活動内容（試験頻出）

  • 1
    プロジェクト憲章の作成 プロジェクトの立ち上げを正式に認可する文書。
  • 2
    プロジェクト計画書の策定 全ての計画（スコープ、コストなど）を統合したマスタープラン。
  • 3
    実行の指揮・管理 計画通りに作業が進むよう全体を指揮する。
  • 4
    変更要求の統合管理 重要 仕様変更などが全体に及ぼす影響を評価し、承認・却下を判断する。
  • 5
    プロジェクトの終結 成果物の引き渡しや、教訓の記録を行い、正式に完了させる。

# 6. 具体例（システム開発の現場で）

機能追加の要望が出た時、統合マネジメント（PM）は板挟みになります。

営業
「機能追加したい！」
開発
「工数が足りない！」
経理
「予算オーバー！」
利用部門
「納期守って！」
PM（統合マネジメント）の判断

「機能追加を受け入れる代わりに、優先度の低い別の機能を次期開発に回して、納期と予算を守ろう」
（スコープ、スケジュール、コストのバランスを調整）

# 7. 他のマネジメントとの違い

管理分野 役割（何を管理するか）
スコープ管理 何を作るか（範囲）
スケジュール管理 いつまでにするか（時間）
コスト管理 いくらでやるか（予算）
品質管理 どのレベルでやるか（品質）
統合マネジメント 全部をまとめて調整・整合性を取る

# 8. ITパスポートでのひっかけ例

❌

「進捗管理だけを行う」

→ ×（それはスケジュール管理。統合マネジメントはもっと広範囲です）

❌

「各管理を個別に最適化する」

→ ×（部分最適ではなく、全体最適が目的です）

❌

「プロジェクトの一部工程のみ対象」

→ ×（立ち上げから終結まで、全期間が対象です）

# 9. ミニ確認問題

Quiz

Q. プロジェクト統合マネジメントの説明として最も適切なものはどれか？

A. 開発作業の品質を管理する
B. 作業スケジュールのみを管理する
C. 各管理分野を横断して全体を調整する
D. 開発担当者のスキルを評価する
👉 正解：C
Aは品質管理、Bはスケジュール管理、Dは資源（人的資源）管理の一部です。全体を横断して調整するのが統合マネジメントです。

まとめ（超短縮）

「部分最適じゃない、全体最適が統合マネジメント」
